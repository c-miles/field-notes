The Symptom

Review & Submit modal

One fast auditor,
one slow network

  • Auditor types in the Auditor Notes textarea. Autosave is armed with a 1 s debounce on input and blur.
  • They finish typing, pause for less than a second, then click Submit. The debounced save is still queued when the completion form fires.
  • The completion form is a plain form_with, so its redirect triggers a full Turbo Visit.
  • From here, timing decides whether Bugsnag screams, or the edits quietly disappear.

Two ways it went wrong

Neither is graceful

Path A · Loud

TurboFrameMissingError

Debounce fires mid-navigation. Notes POST goes out via Turbo. Response arrives on the redirected page, where the target <turbo-frame> no longer exists. Turbo throws.

TRIGGERS WHEN: slow completion POST leaves the race window open. Production conditions (Carmen · 2 s round-trip).
Path B · Silent

Dropped keystrokes

Debounce fires after navigation completes. The isConnected guard in save() catches the detached form, logs, and returns. The user's latest edits never reach the server.

TRIGGERS WHEN: fast completion POST beats the debounce. Every local dev test. No Bugsnag, no telemetry.

The mechanism

Two handlers, one flush

autosave_controller.js 
// Flush pending saves before another form submits or Turbo navigates away.
// turbo:submit-start fires synchronously so we beat the form's network
// round-trip; turbo:before-visit catches link clicks and Turbo.visit.
// Flush via plain fetch so Turbo can't trip a frame-missing error. AUD-665.
handleForeignSubmitStart = (event) => {
  if (!(event.target instanceof HTMLFormElement)) return
  if (this.element.contains(event.target)) return
  if (event.defaultPrevented) return

  this.flushPendingSave()
}

flushPendingSave = async () => {
  const hadPending = this.debounceTimer !== null
  this.clearDebounceTimer()
  if (this.saveInProgress) this.saveRequestId++
  if (!hadPending || !this.element.isConnected || !this.hasFormChanged()) return

  const response = await fetch(this.element.action, {
    method: 'POST', body: new FormData(this.element),
    credentials: 'same-origin', keepalive: true
  })
  if (response.ok) this.captureFormState()  // baseline only on confirmed write
}
turbo:submit-start

Foreign form submissions

Fires synchronously the instant any Turbo form on the page submits. Beats the form's network round-trip, so we cancel pending work before Turbo sees a response to misapply.

Covers
  • Completion form submission (the reported bug)
  • Any cross-form interaction on pages with autosave
  • Same-page frame-update submits on other forms
turbo:before-visit

Direct navigation

Fires when Turbo is about to visit a new URL from a non-form path. Secondary safety net — if the user leaves without submitting anything, we still flush the debounce.

Covers
  • Link clicks on a Turbo-driven <a>
  • Programmatic Turbo.visit() from other controllers
  • Back / forward with pending edits

Four reasons

Fixes two bugs with one mechanism

  • Closes both failure paths. The loud error and the silent data loss resolve through the same flush.
  • Defense in depth. submit-start covers foreign forms, before-visit covers link and programmatic navigation.
  • Idiomatic for the codebase. Async/await plus try/catch matches select_controller.js. No new dependency — project already uses plain fetch.
  • Respects existing semantics. Only fires when there are pending changes, keeps isConnected guard, advances the change-detection baseline only on server confirmation.

Guardrails inside the flush

Fire-and-forget, but careful

  • keepalive: true — the browser finishes the request even as Turbo navigates. Per Fetch spec, payload capped at ~64 KB; autosave forms fit comfortably.
  • captureFormState() only on response.ok — a failed flush leaves the baseline alone, so the next debounce re-sends the same delta.
  • saveRequestId++ when a save is in flight — the existing stale-request guard in save() drops the old Turbo response on our side.
  • this.element.contains(event.target) — prevents the form from flushing in response to its own submission during morph-replacement.
