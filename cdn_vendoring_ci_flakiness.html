CDN Weather

How two shaky CDNs broke every audit CI run recently, and how we could end it by vendoring our JavaScript.

01 · The problem

Every PR went red at once

  • June 30, evening. Run System Tests starts failing on every branch, for every author, no matter what the diff touches.
  • 7 to 18 failures per run, a different set each time. Re-runs just reshuffle which specs die.
  • Always the same family: selects never upgrade, menus never expand, date filters stay empty.
  • The plain RSpec job stayed green the whole time. Only browser specs bleed.

02 · The exposure

How a pin becomes a failure

01 · PIN
A name maps to a URL
importmap.rb points 15 packages at ga.jspm.io and cdn.jsdelivr.net. TomSelect, Floating UI, date pickers, charts.
02 · PAGE LOAD
The browser fetches live
Import maps resolve in the visitor's browser. Every page load downloads the modules from the CDN, right then.
03 · COLD CACHE
Specs never reuse
Each system spec gets a fresh browser context. No HTTP cache. Every page pays the full CDN price again.
04 · THE 3s FUSE
Stall equals red
Capybara waits 3 seconds. A slow or refused fetch means a dead Stimulus controller and a failed assertion.

6 workers × ~590 specs = thousands of live CDN bets per CI run

03 · Findings

The alibi is airtight

  • Main's nightly full suite passed June 30 and failed July 1 on the identical commit. Nothing merged in between.
  • Runner fingerprints were byte identical between green and red. Same Chromium, same postgres image, same caches, same gems.
  • Failure screenshots show fully rendered pages with bare native selects. The HTML arrived. The CDN JavaScript did not.
  • Every failing assertion maps to a controller that imports a CDN module. Locally served JS never missed once.

04 · Two different storms

Both CDNs failed us the same week, differently

Reported outage · the stall
cdn.jsdelivr.net
Documented multi region outage on June 30: "Error 503 No healthy backends" from five countries for about 46 minutes, plus first byte timeouts reported at a Fastly edge the same day. It serves our TomSelect bundle and stylesheet. A stalled fetch outlives the 3 second budget and the spec dies waiting.
Measured rejection · the slam
ga.jspm.io
Our probe, run from inside the runners, caught it live: 5 to 10 percent of connections instantly reset at the TCP level, while github.com and jsDelivr were clean through the same egress. It serves Floating UI, stimulus-use, and cally. A refused module is a dead controller, and no timeout cures that.

05 · Best guess

One noisy neighborhood

Every Blacksmith customer shares the same egress IPs, and anycast routing sends all of it to one CDN edge. Two live suspects, both unconfirmed: an unhealthy CacheFly edge node on our path (ga.jspm.io runs on CacheFly, which scheduled emergency hardware replacement for July 2), or an anti abuse layer limiting our shared address space. Runner images, fleets, our configs, and our test volume are ruled out. The resets reproduce with 20 polite sequential requests.

“
Pinning should just always download.
DHH · importmap-rails v2.0.0 · January 2024

Vendoring has been the Rails default for two and a half years: "you should just serve all your assets, including npm dependencies, off your own domain." Our CDN pins are the old style, inherited from kode eighteen days before the default flipped, and carried forward by convention ever since.

07 · The proposal · PR #928

Serve our own JavaScript

  • PR #928 points all 15 CDN pins at files in vendor/javascript, at the exact versions production already runs. About 750KB, committed once.
  • Zero external URLs remain in the branch's resolved import map. CI would stop caring about CDN weather, and so would production.
  • TomSelect and the two orchidjs packages ship multi file ESM the gem cannot vendor, so those three are bundled by hand from jsDelivr, recipe documented in each file header.
  • The TomSelect stylesheet moves in app for the same reason.

08 · Day two

Owning it would take four commands

terminal 
bin/importmap pin tom-select@2.4.3  # download + write the pin
bin/importmap outdated              # what has newer releases
bin/importmap update                # bump the stale ones
bin/importmap audit                 # CVE check, already in CI

# No CDN pin needed first: one command resolves the package,
# downloads it to vendor/javascript, and writes the pin line.
# Caveat: pristine and update clobber the 3 hand bundled files,
# rebuild those by hand when upgrading (noted in importmap.rb).

09 · Forecast

The weather continues

This was not a one night storm. The suite flaked at a background rate all June, and June 20 and June 23 were early warnings we read as noise. kode still rides the same CDNs behind a bigger timeout. Font Awesome still loads from a kit. Nobody watches pin versions automatically. Vendoring would take down our biggest antenna, not fix the sky.
