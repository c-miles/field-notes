AUD-688 · Engineering Review

Program Eligibility
& Report Page Visibility

Stop surfacing coders who have stopped coding.

The Problem

Removed coders never leave the reporting page.

  • When a coder is removed from a team after enrolling in monthlies, their memberships persist forever.
  • They keep surfacing on the reporting dashboard with empty 0 / 0 / 0 tiles that carry no information.
  • Historical audits must stay intact and re-engagement must still work if the coder comes back.
“
If no chart was received in the last 14 days, pause the program and do not show the tile. If an entire month elapsed with no activity, do not show that month.
Kiley Blake, on AUD-688

The Rule

Three moves, one intent.

01 · Pause
No matching chart in 14 days
A nightly job pauses a membership when its coder has not produced a chart matching the program’s tuple within the window.
02 · Resume
Next matching chart wins
The moment a matching chart is ingested, the membership resumes synchronously. No overnight lag.
03 · Hide
Cadence-aware visibility
Monthly tiles hide whenever the month is empty. Initial and one-off tiles hide only when empty AND paused.

Updated by the recorder on every chart ingest. Read by the pause job every night.

MembershipActivityRecorder

Sync, locked, forward-only.

  • Matched tuple: coder_id, client_id, source, chart_kind_id.
  • with_lock reloads the row under SELECT FOR UPDATE; advance and resume both see fresh state.
  • Monotonic: a late chart never rewinds last_chart_coded_at.
  • Strict >: a chart coded at or before paused_at does not resume. The skip is logged.
  • Per-membership rescue: one bad row cannot poison the batch.
membership_activity_recorder.rb 
def apply(membership)
  membership.with_lock do
    advance_last_chart_coded_at(membership)
    resume_if_eligible(membership)
  end
rescue AuditProgramMembership::InvalidTransition,
       ActiveRecord::RecordInvalid => e
  Rails.logger.error(
    "[MembershipActivityRecorder] Skipping " \
    "membership_id=#{membership.id}: #{e.message}"
  )
end

InactivityPauseJob

Iterable, idempotent, observable.

  • Scope: active.for_standard_programs.idle_since(cutoff). Falls back to enrolled_at when no chart was ever received.
  • One-offs excluded via the standard-programs filter: they are manual-only and never receive chart-driven activity.
  • Reload + recheck inside each_iteration: ingestion or a prior tick may have moved the row.
  • Rescues InvalidTransition and RecordInvalid; one poison row does not cycle the whole job.
  • Open-audit log: pauses with open audits emit an info line so support has a trail.
inactivity_pause_job.rb 
def each_iteration(membership)
  membership.reload
  return unless membership.active?

  reference = membership.last_chart_coded_at ||
              membership.enrolled_at
  return if reference >= cutoff

  membership.pause!
rescue AuditProgramMembership::InvalidTransition,
       ActiveRecord::RecordInvalid => e
  Rails.logger.error("...")
end

Reporting Visibility

One predicate, two cadences.

audit_program_membership_summary.rb 
# Active enrollments stay visible until work appears; monthly tiles hide
# as soon as they are empty.
def visible?
  return true unless empty?
  return false if period_start
  !membership.paused?
end

Applied in Reporting::SummaryQuery via .select(&:visible?). Dashboard, PDF, and XLSX exports all share this filter.

Edge Cases

What the implementation handles.

ScenarioBehaviorSafeguard
Chart coded at or before paused_at Resume does not fire. The skip is logged at info level. log
Concurrent chart ingest vs. pause job Row-level lock serializes the advance and resume check on a fresh reload. with_lock
Deactivated coder produces a chart ChartProcessingJob skips enrollment and the recorder entirely; logged. guard + log
One-off program membership Never paused. Filtered out at the query level since one-offs are manual-only. for_standard_programs
Past month with zero audits Hidden. Monthly tiles drop as soon as empty, pause state aside. visible?
Fresh enrollment, no audits yet Stays visible. Grace period via enrolled_at fallback. idle_since
Membership exactly at cutoff Not considered inactive. Exclusive range: ...cutoff. spec
Paused membership with real audits Still visible. The past is not erased. visible?
Membership raises on pause! Rescued, logged, batch continues with the next row. rescue
Pausing a membership with open audits Emits an info log so support can trace the cross-job race. log

Deploy Safety

Two migrations. One deploy. No operator step.

  • Migration 1: adds the column and concurrent index with safety_assured and disable_ddl_transaction!.
  • Migration 2: backfills from the charts table using stub models defined inside the migration.
  • Drift-proof: the stubs pin to table names, not to current AR classes; future renames or validation changes cannot break a replay.
  • SQL-filtered: only standard memberships iterate via where.not(audit_programs: {chart_kind_id: nil}). On production that drops iteration from 50,655 to 777 rows, sub-5-second run.

Out of Scope, by Choice

What we deliberately did not build.

  • Separate CoderChartActivity table. YAGNI. The summary does not read activity; the pause job would gain a join for no current payoff.
  • Cancel open audits on pause. Different feature, different semantics. Logged instead.
  • Notify coder of pause. Out of scope for this ticket.
  • Structured return value from the recorder. No caller needs it yet; the log trail is sufficient.

Before

Removed coders lingered.

  • Reporting page kept every coder forever.
  • Empty tiles carried zero information but cluttered the view.
  • Exports rendered empty months and empty initial rounds.
  • No operational signal distinguished inactive from active.

After

Inactivity self-heals.

  • Stale memberships pause within a day of crossing 14.
  • Past months with real work stay on the report.
  • Empty months and active-empty tiles drop off per cadence.
  • First matching chart snaps the coder back into view.

Rollout & Monitoring

What ops sees after merge.

  • Events: audit_program_membership.paused and .resumed flow through EventLog as today.
  • Logs: every rescued error, every skipped-resume, every pause-with-open-audits has a named, searchable line.
  • Env flip: ops can widen or tighten the 14-day window without a deploy.
  • Backfill: runs during the same deploy as the column add; no operator coordination required.

TL;DR

Pause per membership after 14 quiet days. Resume on the next chart. Hide empty tiles per cadence.
