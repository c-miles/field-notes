The orchestra AUD-730 · Hiring the Conductor No. 02 / 11

Three pull requests built the players.

AUD-727 added the Stage 1 decision: InternalResolver. AUD-728 added Stage 2: ExternalResolver. AUD-729 composed them into Orchestrator, the class that runs both in sequence and applies their verdicts.

Each is a pure function. Each does what it does when it gets called. None of them know when to run. The full project context lives in the engineering plan deck. This one zooms in on the slice that ties the players to real events.

The gap AUD-730 · Hiring the Conductor No. 03 / 11

An orchestrator is a function. A function isn't a system.

Until something hands the orchestrator a cycle, the gates never run. Auditors finish evaluations in the UI all day long. The cycle never gets created. No verdict gets applied. Nothing surfaces.

This pull request is the listener and the worker that turn evaluation events into cycle decisions. Two new files, two new lines of EventBus wiring. The lift is in what those small additions activate.

The listener AUD-730 · Hiring the Conductor No. 05 / 11

ResolutionHandler is a wire, not a worker.

It subscribes to two events: evaluation.completed and evaluation.discarded. When either fires, it queues the job with the evaluation's id. The work happens in the job, not in the handler.

Kept thin on purpose. Retries, error handling, and database calls all live downstream. The handler exists so nothing in the EventBus path can fail loudly.

The worker AUD-730 · Hiring the Conductor No. 06 / 11

ResolutionHandlerJob in five steps.

1
Load Fetch the evaluation by id. If it was deleted between event emit and pickup, log info-level and skip silently.
2
Filter Skip if the program has no approval_config. One-off and direct-source programs sit outside the workflow.
3
Find or create Race-safe cycle lookup keyed on (audit_program, coder, period_key).
4
Attach Race-safe join row creation linking this evaluation to the cycle.
5
Resolve Hand the cycle to Orchestrator.call(cycle:). Verdicts get applied, events emit, the cycle settles or surfaces.

Verdicts AUD-730 · Hiring the Conductor No. 08 / 11

Eight symbols. Two that act, six that don't.

Stage 1 · InternalResolver
:auto_approve
Above-threshold or post Add More. Orchestrator stamps internal_approved_at and emits auto_approved.
:hold
Below threshold without an Add More request. Cycle surfaces in the system admin's approval queue.
:already_settled
Cycle was internally approved on a previous run. Skip. Stage 2 still runs.
:not_yet_ready
Target not met and period not closed. Cycle keeps gathering evaluations.
Stage 2 · ExternalResolver
:auto_release
Above-threshold and Stage 1 settled. Orchestrator stamps auto_released_at and emits auto_released.
:hold
Below threshold. Cycle waits for an external reviewer to approve manually.
:already_settled
Cycle was previously released. Skip.
:not_yet_ready
Stage 1 hasn't approved yet. Stage 2 doesn't run.
Edge cases AUD-730 · Hiring the Conductor No. 09 / 11

What the pipeline handles.

Concurrent jobs on the same cycle
Race-safe find or create on both the cycle and the join row, backed by unique indexes. RecordNotUnique falls back to find_by!.
Evaluation deleted before the job runs
Initial Evaluation.find raises. The job logs info-level and skips silently. No failure event.
Orchestrator raises unexpectedly
Top-level rescue catches, logs, sends to Bugsnag, emits resolution_failed. No re-raise so Sidekiq doesn't retry and double-emit auto-events.
Program has no approval_config
Job returns early. One-off and direct-source programs sit outside the workflow.
Same evaluation completes then discards
Two events fire, two jobs run. Orchestrator returns :already_settled on the second pass. Idempotent.
Resolver dependency on cycle state
Aggregate score and target progress are derivations, not cached columns. Each orchestrator run re-reads from current evaluations.
Events AUD-730 · Hiring the Conductor No. 10 / 11

Decision moments are events.

On success

auto_approved · auto_released

Emitted by the orchestrator after the matching setter stamps the cycle. Persisted by EventWriter. Future external subscribers compose by listening here.

On rescue

resolution_failed

Emitted by the orchestrator and the job. Same payload shape: cycle_id, audit_program_id, coder_id, error, message.

What activates AUD-730 · Hiring the Conductor No. 11 / 11

When this merges, the workflow goes live.

From this point forward, every evaluation completed in a platform program walks the full pipeline. Cycles get created. Gates run. Above-threshold cycles auto-approve and auto-release without anyone clicking. Below-threshold cycles surface in the System Admin's approval queue (Jill, in our environment) for manual review.

The approval queue page already shipped. Until now it had no real data flowing in. After this merge, it has the actual production stream.
